## 三、获取患者首页序号 / 挂号序号（SQL Server 视图）

用于根据**病历号（住院）或门诊卡号（门诊）**解析出 **xh**（住院为首页序号，门诊为挂号序号），再以 xh 调用上述 LIS/检查接口。

### 3.1 数据库配置

| 配置项 | 值 |
|--------|-----|
| 服务器名 | 10.8.80.21 |
| 用户名 | AISET |
| 密码 | （由院方提供；部署时须放入 Secret 或 .env，勿提交代码库） |
| 库名 | THIS4 |
| 视图名 | 以院方实际为准，如 **VW_BRJZXXK**（见下方视图结构） |
| 字段 | **xh**：首页序号/挂号序号；**pat_num**：病历号/门诊卡号；**jlrq**：记录日期（住院为入院日期 rqrq，门诊为挂号日期 ghrq），用于「近一周或两周」报告范围 |

### 3.2 视图结构（VW_BRJZXXK，含 jlrq）

院方视图需包含 **xh**、**pat_num**、**jlrq**，以便本系统支持「当前时间前一周或两周」的门急诊、住院报告。示例：

```sql
CREATE VIEW [dbo].[VW_BRJZXXK]
/*
视图说明：
- 合并 ZY_BRSYK（住院病人）和 GH_NGHZDK（门诊挂号）两张表的患者编号信息
- 筛选条件：近1年内的记录
- 字段说明：
  xh: 患者序号（住院为首页序号，门诊为挂号序号）
  pat_num: 患者编号（住院为 blh，门诊为 cardno）
  jlrq: 记录日期（住院为 rqrq 入院日期，门诊为 ghrq 挂号日期），供本系统按「近 7/14 天」过滤
*/
AS
SELECT 
    syxh AS 'xh',
    blh AS 'pat_num',
    rqrq AS 'jlrq'
FROM ZY_BRSYK 
WHERE 
    ISDATE(LEFT(ryrq,8) + ' '+ SUBSTRING(ryrq,9,8)) = 1
    AND CONVERT(datetime,LEFT(ryrq,8) + ' '+ SUBSTRING(ryrq,9,8),120) >= DATEADD(YEAR,-1,GETDATE())

UNION ALL

SELECT 
    xh AS 'xh',
    cardno AS 'pat_num',
    ghrq AS 'jlrq'
FROM GH_NGHZDK 
WHERE 
    ISDATE(LEFT(ghrq,8) + ' '+ SUBSTRING(ghrq,9,8)) = 1
    AND CONVERT(datetime,LEFT(ghrq,8) + ' '+ SUBSTRING(ghrq,9,8),120) >= DATEADD(YEAR,-1,GETDATE());
GO
```

本系统配置 **MSSQL_VIEW_REPORT_DAYS=7** 或 **14** 后，仅会取 `jlrq >= (当前日期 - N 天)` 的 xh，再按每个 xh 调 LIS 取报告并合并列表，从而实现「当前时间前一周或两周」的门急诊、住院报告。

## 四、产品对接契合度评估

### 4.1 已契合部分

| 院内接口 | 本系统能力 | 说明 |
|----------|------------|------|
| **入参 HID** | 完全支持 | 卫宁适配器 ASMX 模式：SOAP 入参为 `{"HID":"..."}`，与接口 1、2 一致。 |
| **pat_num → xh 解析** | 完全支持 | 配置 MSSQL（服务器 10.8.80.21、库 THIS4、视图名、xh/pat_num 列名），本系统在收到病历号/门诊卡号时先查视图得 xh，再以 xh 调 LIS。 |
| **视图字段** | 完全支持 | 配置项 `MSSQL_VIEW_HID`、`MSSQL_COLUMN_PAT_NUM`、`MSSQL_COLUMN_XH` 可填院方视图名及列名（如 xh、pat_num）。若实际视图名为 V_Patient_Info，将 `MSSQL_VIEW_HID=V_Patient_Info` 即可。 |

### 4.2 配置对应关系（实施时填写）

- **检验（8092）+ 检查（8091）双端口**  
  - 产品已支持**双 ASMX**：`LIS_API_BASE_URL` 填检验（LisWebService.asmx），`LIS_EXAM_ASMX_BASE_URL` 填检查（TechQueue.asmx）。  
  - 配置示例：  
    - `LIS_API_BASE_URL=http://10.8.80.23:8092/LisWebService.asmx`  
    - `LIS_EXAM_ASMX_BASE_URL=http://10.8.80.23:8091/TechQueue.asmx`  
    - `LIS_USE_ASMX=true`，方法名/入参名按院方文档（通常 `GetReportList`、`jsonParam`）。  
  - 报告列表会**合并**检验与检查；每条带 `report_source`（lab/exam），PDF 代理按此选 8092 或 8091 取流。

### 4.3 注意事项

| 项目 | 说明 |
|------|------|
| **视图名** | 文档中示例视图名为 **V_Patient_Info**，配置默认值为 **VW_BRJZXXK**。以院方实际视图名为准，在配置中设置 `MSSQL_VIEW_HID` 即可。 |
| **敏感信息** | 本文档中的服务器、账号、密码为正式环境信息。部署时务必放入 Secret 或 .env，勿提交到代码库。 |

### 4.4 结论

- **pat_num → xh → LIS** 链路与院内视图、HID 含义**完全契合**，按文档配置 MSSQL + LIS 即可。  
- **检验（8092）+ 检查（8091）** 产品已支持双 ASMX：配置 `LIS_API_BASE_URL` 与 `LIS_EXAM_ASMX_BASE_URL` 后，列表合并、PDF 按 `report_source`（lab/exam）自动选对应接口，**与院内正式接口完美契合**。

## 五、实施配置速查

| 配置项 | 说明 | 示例值（按院方实际修改） |
|--------|------|---------------------------|
| LIS_API_BASE_URL | 检验报告（ASMX 8092） | http://10.8.80.23:8092/LisWebService.asmx |
| LIS_EXAM_ASMX_BASE_URL | 检查报告（ASMX 8091） | http://10.8.80.23:8091/TechQueue.asmx |
| LIS_USE_ASMX | 是否使用 ASMX | true |
| LIS_ASMX_METHOD_NAME | 方法名 | GetReportList（以院方为准） |
| LIS_ASMX_JSON_PARAM_NAME | 入参名 | jsonParam（以院方为准） |
| MSSQL_ENABLED | 是否启用 pat_num→xh | true |
| MSSQL_SERVER | SQL Server 地址 | 10.8.80.21 |
| MSSQL_USER | 数据库用户 | AISET |
| MSSQL_PASSWORD | 数据库密码 | 放入 Secret，勿写进文档 |
| MSSQL_DATABASE | 库名 | THIS4 |
| MSSQL_VIEW_HID | 视图名（xh/pat_num/jlrq） | VW_BRJZXXK（以院方为准） |
| MSSQL_COLUMN_PAT_NUM | 病历号/门诊卡号列 | pat_num |
| MSSQL_COLUMN_XH | 首页序号/挂号序号列 | xh |
| MSSQL_COLUMN_JLRQ | 记录日期列（住院 rqrq/门诊 ghrq） | jlrq |
| MSSQL_VIEW_REPORT_DAYS | 报告范围（天）；0=不按日期只取 TOP1；7/14=近一周或两周门急诊/住院 | 7 或 14 |
